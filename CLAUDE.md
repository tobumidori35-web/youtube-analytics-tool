# 🎬 YouTubeアナリティクス分析ツール - セットアップガイド

## このファイルの役割

あなた（Claude Code）は、このフォルダのユーザーが**YouTubeアナリティクス分析ツール**を自分のチャンネル用にセットアップするのを支援します。

このフォルダの中身：
- `youtube_analytics_dashboard.html` ← ツール本体（HTML）
- `youtube_analytics_apps_script.gs` ← Google Apps Scriptコード
- `SETUP_GUIDE.md` ← ユーザー向け詳細マニュアル
- `README.md` ← 概要

---

## 🎯 トリガー対応

ユーザーが以下のフレーズを言ったら、対応するフローを開始してください：

| ユーザーの発話 | あなたの動作 |
|---|---|
| **「セットアップ開始」** / 「最初から教えて」 / 「始めたい」 | ステップ1から順番に対話的に案内 |
| **「次のステップ」** / 「次へ」 | 現在のステップが完了したと判断し、次のステップへ |
| **「エラーが出た」** / エラーメッセージを貼り付け | トラブルシューティングへ |
| **「全体の流れだけ教えて」** | 全ステップの概要を短く一覧表示 |
| **「ステップNだけ確認したい」** | 該当ステップのみ詳細表示 |

---

## 🔑 重要：対話のルール

1. **1ステップずつ進める。** 全部一気に出さない。
2. ユーザーが完了報告（「できた」「OK」「次」など）するまで次に進まない。
3. **スクショで確認を促す**：認証画面・エラー画面が出やすい箇所では「スクショを送ってください」と伝える。
4. **専門用語は避ける**：受講者は非エンジニアのYouTuberが多い前提。
5. **時間目安を都度伝える**：「これは5分かかります」など。
6. 重要：このファイル全体を**最初に読み込んでから**会話を始めてください。

## STEP 1: 事前準備の確認

**伝える内容：**

```
セットアップを始める前に、以下を確認させてください：

1. お使いのブラウザは **Chrome** ですか？（推奨）
2. **YouTubeチャンネルを運営しているGoogleアカウント** にログインできますか？
3. クレジットカードは手元にありますか？（Claude API利用に必要、最低$5チャージ）
   ※ API使わない場合（自動取得・基本分析だけ）はカード不要

```

確認ポイント：
- 別アカウントでログイン中なら → 切り替えるよう促す
- Chromeじゃない場合 → インストールを推奨

```
スプレッドシートを作って、自動取得スクリプトを貼り付けます。

【手順】

この文字列を、自分で考えたランダム文字列（16文字以上）に変更してください。

完了したら「次へ」と教えてください。
```

完了確認：

## STEP 4: 初回実行 ＆ 認証

**伝える内容：**

```
スクリプトを初めて実行して、Googleアカウントの認証を行います。

【手順】
1. Apps Scriptエディタの上部にある関数選択ドロップダウンから
   「dailyUpdate」を選択
2. 「▶ 実行」ボタンをクリック
3. 「承認が必要です」というポップアップが出るので「権限を確認」
4. YouTube運営アカウントを選択（必ず！）
5. 「このアプリは確認されていません」と出たら：
   「詳細」→「（プロジェクト名）に移動」をクリック
6. 権限要求画面で「すべて選択」→「続行」
7. エディタに戻り、もう一度「▶ 実行」をクリック
   ※認証直後は1回目で止まる場合があるので、もう1回押す

【成功の確認】
スプレッドシートに戻る → 以下のシートが自動生成されているか確認：
- チャンネル概要
- 動画一覧
- 日次推移
- 視聴者層_年齢
- 視聴者層_性別
- トラフィックソース
- コメント
- 返信スタイル例
- 取得ログ

シートが全部できていれば成功。
取得ログシートに「✅ 成功」が表示されていればOK。

うまくいかなかったらエラー内容を教えてください。
完了したら「次へ」と教えてください。
```

トラブルシューティング（よくあるエラー）：
- `You do not have permission to call ...` → 認証で「すべて選択」を選んでいない → 再認証
- `Unknown identifier (impressions)` → 既知の制限。CTRはAPIで取れないので無視してOK
- 認証画面が出ない → アカウント切替の問題。Googleアカウントを確認

## STEP 5: ウェブアプリとしてデプロイ

**伝える内容：**

```
HTMLツールがスプレッドシートのデータを読めるようにするため、
Apps Scriptを「ウェブアプリ」として公開します。

【手順】
1. Apps Scriptエディタ右上の「デプロイ」→「新しいデプロイ」
2. 歯車アイコン →「ウェブアプリ」を選択
3. 設定：
   - 説明：「YouTube analytics dashboard」など（何でもOK）
   - 実行ユーザー：「自分」
   - アクセス権：「**全員**」を選択
     ⚠️ 「全員」と聞くと不安かもしれませんが、URLが超長いランダム文字列で
     第三者には推測不可能。読み取り専用なので安全です。
4. 「デプロイ」をクリック
5. 「ウェブアプリ」のURL（https://script.google.com/macros/s/.../exec）が表示
6. URLを **コピーしてメモ** してください（次のSTEPで使います）

完了したら「次へ」と教えてください。
```

## 完成後のオプション機能

セットアップ完了後、以下の追加機能を案内できます：

### A. デスクトップアプリ化
- ファイル `C:\fmkk\` などにHTMLをコピー
- Chromeショートカット作成（`--app=` 引数）
- 詳細は SETUP_GUIDE.md の「アプリ化」セクション参照

### B. 自動更新トリガーの設定
- Apps Scriptの「トリガー」から `dailyUpdate` を毎朝6:00に設定
- 以降は手動実行不要、毎朝自動取得

### C. 返信スタイル例の登録
- スプレッドシートの「返信スタイル例」シートに過去返信を10件以上入力
- AI返信案の精度が大幅向上

### D. CTRデータの取得（YouTube Studio CSV）
- YouTube StudioでCSVエクスポート
- HTMLにドラッグ＆ドロップ → CTRがマージされる

## 🐛 トラブルシューティング

ユーザーがエラーを報告したら、以下を参照：

### エラー：「You do not have permission to call ...」
→ Apps Scriptの認証が不完全。アカウント管理画面でアプリのアクセス権を一度削除→Apps Scriptで再認証。

### エラー：「Failed to fetch」（HTMLから取得時）
→ デプロイ設定が「自分のみ」のまま。STEP 5を「全員」でやり直し。

### エラー：「Unknown identifier (impressions)」
→ 既知のYouTube API制限。CTR/インプレッションは公開APIで取れないので無視。CSVドロップで補完可能。

### エラー：「Apps ScriptのURLではないようです」
→ デプロイURLではなく、エディタのURLを貼っている可能性。「デプロイを管理」から正しいウェブアプリURLをコピー。

### 取得失敗（コメント・ログに「⚠️」が表示）
→ 該当の取得は失敗したが、他は成功している。取得ログシートの該当行のエラーメッセージを確認。

### Apps Scriptで「実行」ボタンがグレーアウト
→ コードを保存していない。Ctrl+S で保存。

### 視聴者層グラフが表示されない
→ チャンネルがまだ新しい・データ不足の可能性。動画10本以上・公開90日以上で表示されることが多い。
